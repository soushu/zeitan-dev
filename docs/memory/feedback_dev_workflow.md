name: feedback_dev_workflow
description: 開発フロー方針 - 実装→ブラウザ確認→修正ループ、マージはユーザー指示待ち
type: feedback
---

- 実装後は必ずブラウザで動作確認を行う
- 問題があれば修正→ブラウザ確認を問題がなくなるまで繰り返す
- main へのマージはユーザーの明示的な指示があるまで行わない
- develop へのマージもユーザーに報告してから行う
- タスクは優先度が高いものから順に着手する

**Why:** ユーザーが品質を確認してからマージしたいため。自動的にマージしない。
**How to apply:** 全ての開発タスクでこのフローを適用。feature/* ブランチで実装 → ローカルでブラウザ確認 → ユーザーに完了報告 → マージ指示を待つ。
